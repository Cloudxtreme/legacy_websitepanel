WSP Server Internal Application Error when registering server

Translations:

  • tr-TR

Error

When registering a new server in WSP you can receive the following message:

WSP Server Internal Application Error
WSP Server application is accessible, but returned an error. Try opening WSP Server URL in the browser and see the actual error.

Resolution

Add the following code to global server web.config (by default it's located in C:\WINDOWS\Microsoft.NET\Framework\v2.0.50727\CONFIG folder): 

<location path="WebsitePanel Server" allowOverride="true"> 
    <system.web> 
      <trust level="Full" originUrl=""/> 
      <identity impersonate="false"/> 
    </system.web> 
  </location> 
  <location path="WebsitePanel Enterprise Server" allowOverride="true"> 
    <system.web> 
      <trust level="Full" originUrl=""/> 
      <identity impersonate="false"/> 
    </system.web> 
  </location>
