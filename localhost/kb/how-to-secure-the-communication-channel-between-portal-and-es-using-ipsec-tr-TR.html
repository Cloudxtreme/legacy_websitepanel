IPSec kullanarak Portal ile ES arasındaki iletişim kanalını güvenceye alma

Ön Paylaşımlı Anahtar (PSK) Kimlik Denetimi Yöntemiyle IPSec kullanarak Portal ile ES arasındaki iletişim kanalını güvenceye alma:

1.     Yerel Bilgisayar IP Güvenlik İlkeleri filtre listesine yeni filtre ekleyin:

1.     Aynalama = EVET

2.     Protokol = TCP

3.     Kaynak Port = TÜMÜ

4.     Hedef Port = 9002 (veya ES'ye atanmış başka port)

5.     Kaynak Adres = TÜMÜ (veya Portal IP Adresi veya ES IP Adresi)

6.     Hedef Adres = IP Adresim

2.     Yerel Bilgisayar IP Güvenlik İlkeleri işlem filtre listesine yeni filtre işlemi ekleyin:

1.     Güvenlik Yöntemleri = Güvenlik Müzakeresi

2.     Tür = Şifreleme ve Tümleştirme

3.     AH Tümleştirme = Yok

4.     ESP Gizliliği = 3DES

5.     ESP Tümleştirme = SHA1

6.     Anahtar Kullanım Ömrü = 0/0

Not: Filtre İşlem Sihirbazı IP Trafik Güvenliği panelinde Güvenlik Türü = Şifreleme ve Tümleştirme seçilmiş olması halinde parametre 2.1) - 2.6) varsayılan parametrelerdir.

3.     Yeni IP Güvenlik Kuralını Atanan İlke listesine ekleyin:

1.     IP Filtre Listesi = adım 1'de oluşturulan filtre

2.     Filtre İşlemi = adım 2'de oluşturulan filtre işlemi

3.     Kimlik Denetimi Yöntemi = Ön Paylaşımlı Anahtar

4.     Tünel Ayarlama = Yok

5.     Bağlantı Türü = Tümü
Not: Tavsiye edilen Ön Paylaşımlı Anahtar (PSK) uzunluğu en az 25 karakterdir.

4.     Kullanılan protokoller/Güvenlik duvarı hususları:

1.     ISAKMP (Internet Güvenlik İlişkisi ve Anahtar Yönetimi Protokolü), Güvenlik İlişkileri kurma, müzakere etme, değiştirme, değiştirme ve silme prosedürlerini ve paket biçimlerini tanımlar. Tüm uygulamalarda 500 portunda UDP kullanılarak ISAKMP için gönderme ve alma olanağını içermelidir.

2.     ESP (Kapsama Güvenlik Yükü) gizlilik, veri kaynağı denetimi ve bağlantı tümleştirme sağlamak için kullanılabilir.

3.     İletişim kanalını test etme:

4.     Tercih edilen ağ monitörünü kullanarak şifrelenmiş ESP çerçeveleri (şifrelenmiş SOAP mesajları) tarafından takip edilen ISAKMP çerçevelerini (güvenlik müzakereleri) alabilmeniz gereklidir.

5.     Yapılandırmaya/ihtiyaçlarınıza göre başka şifreleme/karma/kimlik denetimi yöntemlerini deneyebilirsiniz:

1.     Şifreleme algoritması: Veri Şifreleme Standardı (DES), Üçlü DES (3DES).

2.     Karma algoritması: MD5 (Mesaj Alma işlevi 5) veya SHA1 (Güvenli Karma Algoritması 1).

3.     Kimlik denetimi yöntemi (Kerberos V5, Sertifika veya ön paylaşımlı anahtar kimlik denetimi).
