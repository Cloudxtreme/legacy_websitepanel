AWStats Kurulum ve Yapılandırma

AWStats Yapılandırma

Bu bölümün amacı aşağıdakilerin nasıl yapılacağını anlatmaktır:

1.       http://stats.customerdomain.com ile AWStats görüntülenmesini sağlama/ ve

2.       AWStats etki alanı istatistiklerine güvenli erişim sağlama

 

Evet, WSP'de varsayılan AWStats hizmet özellikleri şunu verir: "http://stats.[domain_name/]" İstatistik görüntüleme URL modeli, ancak bu AWStats uygulamasının bu URL'ye otomatik yanıt vereceği anlamına gelmez. AWStats uygulamasının her http://stats.customerdomain.com/ URL'sinde yanıt vermesi için iki yol bulunmaktadır. Her iki yöntem, AWStats için özel bir IP adresi olup olmamasına bağlıdır.

 

AWStats özel IP'de

1. AWStats uygulamasını ayrı IIS web sitesine kurun.

2. AWStats web sitesini özel IP'ye bağlayın. AWStats web sitesi IIS yöneticisinde gerekli IP ve portu (varsayılan 80) "Web Sitesi" sekmesi -> "Web Sitesi Tanımlama" bölgesi -> "Gelişmiş..." iletişim kutusunda ekleyin.

3. WSP uygulamasında, "DNS Kayıtları" bölümü AWStats hizmet özellikleri sayfasına yeni "istatistik" (veya başka özel ad) "A" kaydı ekleyin ve AWStats özel IP'si ile sağlayın.

 

Her müşteri DNS bölgesinde "istatistik" "A" kaydı bulunacak ve tarayıcıda http://stats.customerdomain.com/ istendiğinde AWStats web sitesi açılacaktır.


AWStats paylaşımlı IP'de

1. Bu durumda AWStats herhangi bir konuma kurulabilir; örneğin "Varsayılan Web Sitesi" "AWStats" sanal dizini.

2. "İstatistikler" "A" DNS kaydı "Varsayılan Web Sitesi" tarafından herhangi bir IP adresini gösterecek şekilde tanımlanabilir (varsayılan olarak tüm IP adreslerine bakar). Ancak WEB hizmetinde tanımlanmış bir "*" "A" kaydı bulunuyorsa, bilinmeyen ana bilgisayar başlıklı (sunucu bağlama) tüm istekler "Varsayılan Web Sitesi" tarafından sunulacağından "istatistik" kaydı olmadan ayrılabilirsiniz.

Her iki özel ve paylaşımlı IP durumunda sonraki adım, gelen http://stats.domain.com/ isteklerinin Hata! Köprü başvurusu geçerli değil. konumuna yönlendirilmesidir. Bunun için şu basit ASP.NET komut dizisini kullanabilirsiniz:

<script runat="server" language="C#">

 void Page_Load()

 {

  Hashtable redirects = new Hashtable();

 

  // belirtilen alt etki alanı bulunmadığında görüntülenmesini istediğiniz metin

  string missingText = "<h2>There is no subdomain here.</h2>";

 

  // Tekrar yönlendirme listesi tanımla.

  // Tekrar yönlendirme ekleme:

  //

  // redirects[<sub-domain name>] = new SubDomainRedirect(

  //  <page title>,

  //  <page URL> - yeri alınacak olan URL'de [domain] değişkenini kullanabilirsiniz,

  //  <true - open URL in the frame; false - perform redirect to the URL>);

  //

  //////////////////////////////////////////////////////

  

  redirects["stats"] = new SubDomainRedirect(

   "Statistics Server",

   "http://awstats.myserver.com/cgi-bin/awstats.pl?config=[domain]",

   true);

   

  redirects["mysql"] = new SubDomainRedirect(

   "PHP MyAdmin",

   "http://mysql.myserver.com",

   true);

   

  // process redirects

  string domain = Request.Url.Host;

 

  int dotIdx = domain.IndexOf(".");

  string subDomain = (dotIdx != -1) ? domain.Substring(0, dotIdx).ToLower() : null;

 

  if (subDomain == null)

  {

   // show missing text

   RenderMissingText(domain, missingText);

  }

  else if (redirects[subDomain] == null)

  {

   // show missing text

   RenderMissingText(domain, missingText);

  }

  else

  {

   SubDomainRedirect redirect = (SubDomainRedirect)redirects[subDomain];

 

   // subsitute domain

   redirect.Url = redirect.Url.Replace("[domain]", domain.Substring(dotIdx+1));

 

   if(!redirect.Framed)

    Response.Redirect(redirect.Url, true);

 

   string title = (redirect.Title != "") ? redirect.Title : domain;

 

   // show frames

   RenderFrames(title, redirect.Url);

  }

 }

 

 private void RenderMissingText(string title, string missingText)

 {

  Response.Write(String.Format(@"<html>

    <head><title>{0}</title>

    <body>

     {1}

    </body>

    </html>", title, missingText));

 }

 

 private void RenderFrames(string title, string url)

 {

  Response.Write(String.Format(@"<html>

    <head><title>{0}</title>

    <frameset rows=""1,*"" border=""0"">

    <frame src="" mce_src=""#"">

    <frame src="" mce_src=""{1}""></frame>

    </frameset>

    </html>", title, url));

 }

 

 public class SubDomainRedirect

 {

  public string Url;

  public string Title;

  public bool Framed;

 

  public SubDomainRedirect(string title, string url, bool framed)

  {

   this.Title = title;

   this.Url = url;

   this.Framed = framed;

  }

 }

</script>

 

"Default.aspx" sayfasını oluşturun, kodu yapıştırın ve paylaşımlı IP için "c:\inetpub\wwwroot" klasörüne, özel IP için AWStats web sitesi kök klasörüne kaydedin.

 

NOT: AWStats Viewer çözümünü kullanıyorsanız (AWStats Uygulamasını Güvene Alma) bu sayfa gerekli değildir.

 

AWStats Uygulamasını Güvene Alma

AWStats tasarım olarak güvenli bir çözüm değildir ve Internet'teki herkes http://stats.customerdomain.com/ adresi aracılığıyla istatistikleri görebilir.

 

AWStats uygulamasını güvene almak için basit bir ASP.NET uygulaması kurma ve yapılandırma (ikililer ve kaynaklar dahil) konusunda bilgi almak için: AWStats Uygulamasını Güvene Alma.
