Active Perl çalışma zamanını ISAPI olarak Microsoft IIS 7.0 üzerine kurma ve çalıştırma

Özet

Microsoft Internet Bilgi Hizmetleri 7.0 (x86 ve x64) üzerinde ActivePerl çalışma zamanı desteği sağlamak için, sunucu üzerinde MSI dağıtım dosyasıyla Active Perl uygulamasını kurun, WebsitePanel ile Internet Bilgi Hizmetleri 7.0 hizmet sağlayıcıyı güncelleyin ve yeni bir barındırma planı ayarlayın veya mevcut birini güncelleyerek Active Perl çalışma zamanını müşteriler için ulaşılabilir kılın.

 

Active Perl Kurma

Active Perl çalışma zamanını kurmak için http://www.activestate.com/activeperl/downloads/ adresini ziyaret edin ve listedeki en güncel sürümü yükleyin. Platformun x86 veya x64 olmasından bağımsız olarak Active Perl uygulamasının yalnızca x86 dağıtım dosyası sürümü her iki platformda kullanılmalıdır.

 

1. Yüklenen dağıtım dosyasını açarak kurulumu çalıştırın

 

İpucu: bileşen bu işlemi gerçekleştirildiğinden Perl modülleri ve kitaplıklarını güncelleme işlemini düzenlemek için PPM bileşenini etkinleştirebilirsiniz.

2. Ekrana gelen tüm seçeneklerin işaretini kaldırın (WebsitePanel komut dosyası uzantıları eşlemesini kendisi uygular) ve İleri düğmesini tıklatın

3. Kurulum işlemi tamamlandığında Perl çalışma zamanının beklenen şekilde çalışıp çalışmadığı test edilmelidir. Öncelikle tüm web sunucusu rolü hizmetlerinde CGI rolü hizmetinin kurulu olduğu onaylanmalıdır (ekli ekran görüntüsüne bakın)

4. Ardından Active Perl çalışma zamanını test etmek için bir demo web sitesi oluşturun (yerel ana sayfa > Siteler > Web Sitesi Ekle)

5. PerlEx30.dll dosyası için IIS'de (localhost > IIS > ISAPI and CGI Restrictions), Active Perl komut satırı çalışma zamanının .pl veya .cgi sayfası istendiğinde web sitesinde kullanılması için bir izin kuralı kaydedin

6. .pl dosya uzantısı eşlemesini perlex30-demo.org web sitesine kaydedin (yerel ana bilgisayar > Siteler > perlex30-demo.org > IIS > Yönetici Eşlemeleri)

7. .cgi dosya uzantısı eşlemesini perlex30-demo.org web sitesine kaydedin (yerel ana bilgisayar > Siteler > perlex30-demo.org > IIS > Yönetici Eşlemeleri)

8. Notepad.exe uygulamasını açın, aşağıdaki komutu yapıştırın ve dosyayı perltest.pl olarak perlex30-demo.org kök klasörüne kaydedin

$url = "http://$env%7bserver_name%7d$env%7burl%7d/";

$ip = "$ENV{REMOTE_ADDR}";

print <<ENDOFTEXT;

HTTP/1.0 200 OK

Content-Type: text/html

&lt;HTML>

<HEAD><TITLE>Hello World!</TITLE></HEAD>

<BODY>

<H4>Hello World!</H4>

<P>You have reached <a href="$url">$url</a></P>

<p>Your IP address is $ip</P>

<H5>Have a nice day!</H5>

</BODY>

</HTML>

ENDOFTEXT

exit(0);

 

9. Bir web tarayıcı açın, web sitesinin url'sini yazın, her şey doğru yapılandırıldığında aşağıdaki sayfa görüntülenmelidir

 

WebsitePanel ayarlarını güncelleme

Internet Bilgi Hizmetleri 7.0 hizmet sağlayıcı ayarlarını güncellemek için tarayıcıda denetim masası portalını açın ve serveradmin olarak oturum açın. Oturum açtıktan sonra Yapılandırma > Sunucular > Web Sunucusu > IIS 7.0 sağlayıcı ayarlarını açın.

 

1. Varsayılan olarak Perl yürütme dosyası yolu ayarı (Web Uzantıları bölümü), varsayılan Perl kurulumu yolunu kullandığınızı varsayar; ancak varsayılan Perl kurulum yolunu değiştirmeniz halinde bu ayarı uygun şekilde güncelleyin

2. Hizmet sağlayıcı ayarları güncellendikten sonra mevcut barındırma planı kotalarını güncelleyin veya Perl özelliği etkinleştirilmiş yeni bir plan oluşturun

3. Müşterileriniz artık, Perl komut satırını bir web sitesinde etkinleştirerek veya devre dışı bırakarak, denetim masası arabirimi aracılığıyla Active Perl çalışma zamanı olanaklarından faydalanabilir
