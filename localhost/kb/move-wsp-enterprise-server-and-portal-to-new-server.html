Move WSP Enterprise Server and Portal to new server

Translations:

  • tr-TR

In order to move WSP Enterprise Server and Portal to new server the following steps could be done:

1. Install WSP Enterprise Server (ES) on Server 2 (destination server).

2. Install WSP Portal on Server 2 and connect portal to ES on Server 2.

3. Backup WSPEnterpriseServer database on Server 1 and restore it on Server 2. Alternatively, if ES database leaves on a dedicated server omit this step.

4. Recreate login for WSP Enterprise database with the same sql password from web.config from Server1 (delete existent login before). Map new login on restored database and make sure that it has correct roles: db_owner and public.

5. Copy Enterprise Server's web.config from Server 1 to Server 2. This file is located in the root of ES application (c:\websitepanel\enterprise server) and contains crypto key with database connection string.

6. In ES web.config adjust database connection string if required.

7. If WSP Server access URL on server 1 is a local "loopback" IP (127.0.0.1) then change it to LAN IP.

8. Login into control panel installed on Server 2, open WSP Server properties page and change access URL.
