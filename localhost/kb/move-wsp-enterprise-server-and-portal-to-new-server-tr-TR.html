WSP Enterprise Server ve Portalı Yeni Sunucuya Taşıma

WSP Enterprise Server ve Portalı yeni sunucuya taşımak için aşağıdaki adımlar uygulanabilir:

1. WSP Enterprise Server (ES) uygulamasını Sunucu 2'ye (hedef sunucu) kurun.

2. WSP Portal uygulamasını Sunucu 2'ye kurun ve portalı Sunucu 2'de ES'ye bağlayın.

3. WSPEnterpriseServer veritabanını Sunucu 1'e yedekleyin ve Sunucu 2'ye geri yükleyin. Alternatif olarak, ES veritabanı özel bir sunucuda bulunuyorsa bu adımı göz ardı edin.

4. Sunucu 1 web yapılandırmasındaki sql şifresiyle yeni bir WSP Enterprise veritabanı oluşturun (mevcut oturum bilgilerini silin). Yeni oturumu geri yüklenen veritabanı ile eşleyin ve rollerin doğru olduğundan emin olun: db_owner and public.

5. Enterprise Server web.config dosyasını Sunucu 1'den Sunucu 2'ye kopyalayın. Bu dosya ES uygulaması kök klasöründe (c:\websitepanel\enterprise server) bulunur ve veritabanı bağlantı dizisini içeren şifre anahtarına sahiptir.

6. Eğer gerekiyorsa, ES web.config dosyasında veritabanı bağlantı dizisini ayarlayın.

7. Eğer Sunucu 1'deki WSP Server erişimi yerel bir "geri döngü" IP'si (127.0.0.1) ise, LAN IP'si olarak değiştirin.

8. Sunucu 2'de kurulu denetim masasında oturum açın, WSP Server özellikler sayfasını açın ve erişim URL'sini değiştirin.
