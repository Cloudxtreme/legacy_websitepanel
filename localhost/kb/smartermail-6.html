SmarterMail 5.x Kurulum ve Yapılandırma

 

Harici bağlantılar:

  • SmarterMail Kurulumu http://help.smartertools.com/SmarterMail/v5/
  • IIS Sitesi olarak SmarterMail Kurulumu http://help.smartertools.com/SmarterMail/v5/
  • IIS Sanal Dizini olarak SmarterMail Kurulumu http://help.smartertools.com/SmarterMail/v5/

 

Kurulum

SmarterMail, ürünü çalıştırmak için gereken her şeyi içeren tek bir kurulum dosyası olarak gelir. SmarterMail kurulum aracı http://www.smartertools.com adresinden yüklenebilir. SmarterMail ücretsiz ve lisanslı sürümü bu kurulum dosyasında bulunur, bu nedenle ücretsiz sürümü denedikten ve satın almaya karar verdikten sonra dosyanın tekrar yüklenmesi veya kurulması gerekli olmayacaktır.

 

Not: Windows 2003 Advanced Server kullanıcılarının, dosya izinlerinin uygun şekilde ayarlanabilmesi için SmarterMail uygulamasını denetim masasında Program Ekle/Kaldır öğesinden kurması gereklidir.

 

IIS 6.0 veya daha yeni bir sürüm varsa, performans nedenleriyle SmarterMail web arabiriminin dahili web sunucusu yerine IIS ile çalıştırılması tavsiye edilir. Daha fazla bilgi almak için IIS sitesi olarak çalıştırma başlığına bakın.

 

IIS ile Web Arabirimini Çalıştırma

Varsayılan olarak SmarterMail, kurulum sonrasında programı hemen kullanmanızı mümkün kılan temel bir web sunucusu kuracaktır. Ancak SmarterMail uygulamasını, Microsoft Internet Bilgi Sunucusu (IIS) gibi daha sağlam ve güvenli bir web sunucusuna taşımanız tavsiye edilir. SmarterMail programını IIS 5.0 veya daha üzeri ile çalışacak şekilde yapılandırmak için, aşağıdaki adım adım talimatları uygulayın.

 

Bu bölümde SmarterMail programının kendi IIS sitesi olarak kurulması anlatılmaktadır (tavsiye edilen yöntem). SmarterMail programını mevcut bir IIS sitesinde sanal dizin olarak çalıştırmak istiyorsanız burayı tıklatın.

 

Not: Bu başlıkta, IIS ve nasıl çalıştığı konusunda bilgi sahibi olduğunuz varsayılmaktadır. SmarterTools, IIS kullanmak istememeniz halinde SmarterMail ile birlikte verilen web sunucusunu kullanmanız tavsiye edilir.

 

SmarterMail Web Sunucusunu Devre Dışı Bırakma

Kesinlikle gerekli olmamakla birlikte bu adım, sunucunun güvenliğini ve performansını artırır ve tavsiye edilir.

 

Bu adımlar SmarterMail ile birlikte verilen Web Sunucusunu durduracak ve devre dışı bırakacaktır. Yeni SmarterMail URL'sini güncelleyene kadar masaüstündeki SmarterMail kısayolunun devre dışı kalması şeklinde bir yan etki de söz konusu olacaktır.

 

Başlat -> Program Dosyaları -> SmarterTools -> SmarterMail -> SmarterMail Web Sunucusunu Yapılandır öğesini tıklatın

Durdur düğmesini tıklatın

Çalıştırma Modunu Devre Dışı olarak değiştirin ve Uygula düğmesini tıklatın

Kapat düğmesini tıklatın

 

Bir Uygulama Havuzu ekleyin (yalnızca Windows 2003)

Başlat > Denetim Masası > Yönetim Araçları > Internet Bilgi Hizmetleri (IIS) öğesini tıklatın

Uygulama Havuzu öğesini sağ tıklatın ve Yeni -> Uygulama Havuzu öğesini seçin

pool SmarterMail (veya eşdeğeri) öğesini adlandırın ve Tamam öğesini tıklatın


SmarterMail uygulamasını IIS'ye ekleyin

Başlat > Denetim Masası > Yönetim Araçları > Internet Bilgi Hizmetleri (IIS) öğesini tıklatın

Sayfanın sol tarafındaki web siteleri ağaç görünümünü sağ tıklatın ve Yeni -> Web Sitesi öğesini seçin

Açıklama istendiğinde SmarterMail girin

 

Webmail için kullanılacak IP adresini girin. Eğer bu IP adresi başka bir web sitesi ile paylaşılıyorsa, farklı bir port veya Ana Bilgisayar Başlığı kullanmanız gerekecektir. Ana Bilgisayar Başlığı hakkında daha fazla bilgi sahibi olmak için IIS belgelerine bakın.

 

Yol için "C:\Program Files\SmarterTools\SmarterMail\MRS" kullanın (veya SmarterMail kurduğunuz konumu kullanın).

 

"Bu web sitesine anonim erişime izin ver" onay kutusunu işaretli olarak bırakın.

Hangi izinlerin verileceği sorulduğunda yalnızca Okuma ve Çalıştırma Komut Dizilerini seçin.

Sihirbazı bitirin

 

Yeni IIS sitesinin web sitesi ağaç görünümünü açın ve posta işleme dizinini sağ tıklatın.

Açılır menüden Özellikler öğesini seçin.

 

İzinleri Yürüt açılır listesinde "Hiçbiri" öğesini seçin ve Tamam öğesini tıklatın

(Yalnızca Windows 2003) Site adını sağ tıklatın ve özellikleri seçin

(Yalnızca Windows 2003) Ana dizin sekmesinde SmarterMail uygulama havuzunu seçin

 

IIS Ayarlarını Doğrulama

Eğer Windows 2003 kullanıyorsanız, ASP.NET v2.0.50727 Web Hizmeti Uzantısında IIS için izin Verilmiş olarak ayarlandığını doğrulayın. Eğer ASP.NET öğesi Web Hizmeti Uzantıları altında görülmüyorsa, ASP.NET öğesinin Uygulama Sunucusunda kurulu olduğundan emin olun. (Denetim Masası -> Program Ekle/Kaldır -> Windows Bileşenleri -> Uygulama Sunucusu -> Ayrıntılar)

 

Site varsayılan belgesinin Default.aspx olduğundan emin olun (Belgeler sekmesinde)

Daha yüksek performans için, yeni skin oluşturmuyorsanız HTTP içeriği kullanma süresini App_Themes klasöründe 7 gün olarak ayarlayın

 

Windows 2003 uygulamasında, uygulama havuzu performans sekmesini "Şu süre boşta kaldığında çalışma işlemini kapat" öğesini devre dışı bırakacak şekilde değiştirin veya 240 gibi yüksek bir değer ayarlayın.

 

DNS Güncelleme

SmarterMail ile herhangi bir etki alanı ayarlanmışsa, webmail DNS kayıtlarını yeni IP adresini gösterecek şekilde güncelleyin.

 

Siteyi Test Etme

Bir web tarayıcı açın ve eklediğiniz sitenin IP adresini girin.

 

Mevcut IIS Sitesi ile Web Arabirimini Çalıştırma

SmarterMail programını IIS 5.0 veya daha üzeri ile mevcut web sitesinde çalışacak şekilde yapılandırmak için, aşağıdaki adım adım talimatları uygulayın.

 

Bu bölümde SmarterMail programını mevcut bir IIS sitesinde sanal dizin olarak ayarlama anlatılmaktadır. Bu, tavsiye edilen yöntem değildir. Uygun şekilde çalışacaktır, ancak SmarterMail programını kendi ISS sitesi olarak ayarlamanız tavsiye edilir. Not: SmarterMail 1.x için gerekli olan bazı adımlar artık gerekli değildir ve bu başlıktan çıkarılmıştır.
