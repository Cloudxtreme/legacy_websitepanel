Using bulk import tool

Translations:

  • tr-TR

Bulk Import Tool helps you to create Exchange mailboxes, Exchange contacts and AD users from the input CSV file. This tool creates objects in Active Directory, Exchange and WSP.

Note that bulk import tool's behaviour is connected to the internal WSP Enterprise Server SQL database structure. While Enterprise Server database structure may be changed from version to version, ensure you get tool version compatible with your current WSP version.

Setup instructions:

1. Create new folder on the server with Enterprise Server component installed (e.g. "BulkImport")

2. Unpack attached zip to the new folder.

3. Open "WSP.EnterpriseBulkImport.exe.config" using notepad and edit the following settings:

   a. ES.WebService - url of the Enterprise Server web service (e.g. "http://127.0.0.1:9002/"). Try to open this url in the browser before to avoid errors.

   b. ES.Username - username of the administrator account (e.g. "serveradmin")

   c. ES.Password - password of the administrator account (e.g. "password")

   d. ConnectionString - ES database connection string. You can copy connection string from "installer.exe.config" located in the "c:\Program Files\SMB SAAS Systems\WebsitePanel Installer\" folder:

<?xml version="1.0" encoding="utf-8" ?>

<installer>  

<components>

<component id="xxx">

   <settings>

   ...

   <add key="ComponentName" value="Enterprise Server" />

   <add key="ConnectionString" value="server=(local)\SQLExpress;database=WSPEnterpriseServer;uid=WSPEnterpriseServer;pwd=nfwfakmn0z7yl4hlrctm;" /

...

   </settings>

</component>

</components>

</installer>

Import instructions:

1. Create new Hosted organization where new mailboxes and contacts to be created (e.g. organization name "Test Organization", organization id "test")

2. Open "WSP.EnterpriseBulkImport.exe.config" using notepad and edit the following settings:

   a. InputFile - name of the csv file (e.g. "test.csv"). Csv file format should be the same as "test.csv" in the attached zip!!!

   b.TargetOrganization - id of the organization where new mailboxes to be created (e.g. "test")

3. Make full backup of WSP Enterprise Server database!

4. Run "WSP.EnterpriseBulkImport.exe" tool

5. Check is imported objects appear in WSP

6. Repeat 2-5 for each organization.
