Toplu içe aktarma aracını kullanma

Toplu İçe Aktarma Aracı, giriş CSV dosyasından Exchange posta kutuları, Exchange kişileri ve AD kullanıcıları oluşturmanıza yardımcı olur. Bu araç, Active Directory, Exchange ve WSP'de nesneler oluşturur.

 

Toplu içe aktarma aracının davranışı, dahili WSP Enterprise Server SQL veritabanı yapısına bağlıdır. Enterprise Server veritabanı yapısı sürümler arasında farklılık gösterse de mevcut WSP sürümünüzle uyumlu araç sürümünü aldığınızdan emin olun.

 

Kurulum talimatları:

1. Enterprise Server bileşeni kurulu halde sunucuda yeni bir klasör oluşturun (örneğin, "BulkImport")

2. Ekteki zip dosyasını yeni bir klasöre çıkarın.

3. "WSP.EnterpriseBulkImport.exe.config" dosyasını not defterini kullanarak açın ve aşağıdaki ayarları düzenleyin:

a. ES.WebService - Enterprise Server web hizmetinin url adresi (örn, "http://127.0.0.1:9002/"). Hataları önlemek için bu url adresini önce tarayıcıda açın.

b. ES.Username - yönetici hesabının kullanıcı adı (örn, "serveradmin")

c. ES.Password - yönetici hesabının şifresi (örn, "password")

d. ConnectionString - ES veritabanı bağlantı dizisi. "c:\Program Files\SMB SAAS Systems\WebsitePanel Installer\" klasöründeki "installer.exe.config" dosyasından bağlantı dizisini kopyalayabilirsiniz:

<?xml version="1.0" encoding="utf-8" ?>

<installer>

<components>

<component id="xxx">

<settings>

...

<add key="ComponentName" value="Enterprise Server" />

<add key="ConnectionString" value="server=(local)\SQLExpress;database=WSPEnterpriseServer;uid=WSPEnterpriseServer;pwd=nfwfakmn0z7yl4hlrctm;" /

...

</settings>

</component>

</components>

</installer>

 

İçe aktarma talimatları:

1. Yeni posta kutuları ve kişilerin oluşturulacağı yeni bir Barındırılan kurum oluşturun (örn, kurum adı "Test Organization", kurum kimliği "test")

2. "WSP.EnterpriseBulkImport.exe.config" dosyasını not defterini kullanarak açın ve aşağıdaki ayarları düzenleyin:

a. InputFile - csv dosyasının adı (örn, "test.csv"). Csv dosya biçimi, ekteki zip dosyasındaki "test.csv" gibi olmalıdır!!!

b.TargetOrganization - yeni posta kutularının oluşturulacağı kurumun kimliği (örn, "test")

3. WSP Enterprise Server veritabanının tam yedeğini alın!

4. "WSP.EnterpriseBulkImport.exe" aracını çalıştırın

5. İçe aktarılan nesnelerin WSP'de görünüp görünmediğini kontrol edin

6. Her kurum için adım 2-5'i tekrarlayın.
