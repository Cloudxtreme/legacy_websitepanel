Tuning Outlook 2003 in WSP HES environment

Translations:

  • tr-TR

Outlook 2003 does not support Autodiscover. That’s why the best way to simplify Outlook 2003 setup is create Outlook Profile (PRF) file.

To read more about Outlook Profile (PRF) files take a look here http://technet2.microsoft.com/Office/en-us/library/71b4efb2-539a-49eb-935e-3222aed9f6a21033.mspx?mfr=true

In case you have no time or desire to dig into this –  upload sample PRF file prepared by us and follow those instructions:

For creating provider-specific template –

  • Open in notepad

  • Change once:

- ProfileName=Hoster_name Exchange Hosting – something pretty to represent provider

  • Change twice:

- HomeServer=exch01.provider.local – internal name of Mailbox server

- RPCProxyServer=exchange.provider.com – external Client Access Server URL

For creating end-user-specific template –

  • Open in notepad

  • Change twice:

- MailboxName= user@customer_domain.com – end user primary e-mail address

Run this file on end-user machine. In case use with Outlook 2007 (may be you can not use Autodiscover for some reason) not simple run but use outlook.exe /importprf with appropriate path to prf file. Run this from folder where outlook.exe located.

Note that Outlook should be closed before running PRF file on client machine. Instruct your customers to check it by looking to Outlook.exe process in Task Manager.

Note that you can use Mailbox Setup Instructions letter to send users link to upload prf file.
