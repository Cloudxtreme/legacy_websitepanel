Setup Exchange Hosting Mode Services

Translations:

  • tr-TR

Step 1: Add New Server

In order to control Exchange 2010, “WebsitePanel Server” agent must be installed on the machine with Exchange “Client Access Server” (CAS) role. WebsitePanel Server application pool identity (“WSServer” by default) must be Active Directory account which is a member of “Enterprise Admins” group.

Login as Server Admin, go to Configuration – Servers and create a new Server.

Step 2: Create Services

On the Server, add service for each of Exchange Service Plan under Exchange Hosting Mode Service Group:

 

For the Name field, use meaningful name like [Program ID,Offer ID]. To find out Program ID and Offer ID access your CAS server and see Service Plans here:
<Exchange Installation Path>/Exchange Server/V14/ClientAccess/ServicePlans
There is a number of SERVICEPLAN files, and they all are listed in ServicePlanHostingRemap.csv.

In the example shown on screenshot above, you would need to create 3 Services with the following Program ID and Offer ID:

HostingSample,3
HostingSample,4

Step 3: Create Virtual Servers

For each of the services created, create a Virtual Server.

Go to Configuration -> Virtual Servers and Click “Add Virtual Server”.

Give Virtual Server meaningful name (for example the same name of for the Service that will be added to this virtual server):

Click “Add Server”. On the Edit Virtual Server screen click “Add Services”.

On Add Virtual Server Services screen select OS Service (in the example below it’s Windows Server 2008) and Service for the Service Plan (in this example it’s HostingSample,2). OS Service should be setup in advance and may be on the same or on different physical server.

Click “Add Selected” button.

Repeat these steps for each Service Plan. In the end there will be a set of Virtual Servers like this one:

Now all services are set up and admin or reseller may create Hosting Plans in WebsitePanel.
