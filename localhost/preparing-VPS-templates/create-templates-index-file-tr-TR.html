Belgeler > VPS Şablonları Hazırlama

Şablon .VHD dosyalarının bulunduğu OS Şablonları klasörü (örneğimizde “C:\VPS\Templates”), kullanılabilir şablonların listesini içeren ve aşağıdaki yapıdaki “index.xml” dosyasını içermelidir:

<?xml version="1.0"?>

<items>

  <item path="vhd-filename-without-extension"

        diskSize="internal-vhd-size-in-GB"

        legacyNetworkAdapter="true|false"

        remoteDesktop="true|false">

    <name>template name shown in the dropdown</name>

    <description>template description</description>

    <provisioning>

      <sysprep file="path-inside-vhd-1"/>

      <sysprep file="path-inside-vhd-1"/>

      ...

      <sysprep file="path-inside-vhd-N"/>

      <vmconfig computerName="true|false"

                administratorPassword="true|false"

                networkAdapters="true|false" />

    </provisioning>

  </item>

  <item path="item2">

      ...

  </item>

  <item path="item3">

      ...

  </item>

  ...

</items>

  • path – VHD dosyasının .VHD uzantısı olmadan adıdır, örneğin “Windows Server 2008 x64”. Bu öznitelik zorunludur.
  • diskSize – VHD'nin gigabyte cinsinden dahili boyutudur (VHD'deki ilk disk biriminin boyutu). Bu değer VPS oluşturma sırasında kullanılır. Eğer VPS için tahsis edilmiş “HDD” kotası belirlenen VHD boyutundan az ise, VPS oluşturulmayacak ve hata görüntülenecektir. Ancak bu öznitelik opsiyoneldir ve belirlenmemesi veya 0 (“sıfır”) olması halinde, VPS oluşturma sırasında disk boyutu onaylanmayacaktır.
  • legacyNetworkAdapter – sanal bilgisayarda oluşturulacak ağ adaptörü türünü belirler: synthetic (“Tümleştirme Hizmetlerinin” bulunduğu yeni Microsoft işletim sistemleri için) veya legacy (Windows XP öncesi Microsoft işletim sistemleri veya Linux gibi Microsoft harici işletim sistemleri için). "legacyNetworkAdapter" özniteliği belirlenmediğinde, varsayılan değer "synthetic" şeklindedir.
  • remoteDesktop – ziyaretçi OS'de uzak masaüstünün etkinleştirilmiş ve VPS'ye Uzak Masaüstü Web Bağlantısı (RDWC) ile erişim sağlanabiliyor olup olmadığını belirler. Eğer remoteDesktop değeri “true” şeklinde ise RDWC bulunan bir bağlantı VPS özellikleri “Genel” sekmesinde görüntülenecektir. Bu öznitelik belirlenmediğinde, varsayılan olarak RDWC bağlantısı gizlenecektir:
  • name – OS şablonunun ekran adıdır. Bu, VPS oluşturma sihirbazındaki açılır listede gösterilen değerdir.
  • description – OS şablonunun açıklamasıdır. Şu anda kullanılmamaktadır.
  • provisioning – denetim seçeneklerini içeren opsiyonel öğedir.
  • sysprep – VHD içinde VHD denetimi sırasında işlenmesi gereken bir dosya belirler. Dosya, DNP şablonu değişkenlerini ve talimatlarını içerebilir. DNP yalnızca ilk VHD birimine erişim sağladığından “dosya” özniteliği göreli yolu (sürücü harfi olmadan) VHD içindeki dosyaya göre belirlemelidir, örneğin “\Windows\System32\sysprep\unattend.xml”. Birden fazla dosyanın işlenmesi için herhangi bir sayıda “sysprep” öğesi olabilir.
  • vmconfig – VPS denetimi sırasında hangi VmConfig hizmet modüllerinin yürütüleceğini belirleyen opsiyonel öğedir. Denetim aşamasında VmConfig kullanılması, eklenmemiş kurulum yanıt dosyasında bazı ayarların yapılandırılmasının zor, hatta olanaksız olduğu durumlarda faydalı olabilir. Örneğin, Windows XP şablonunda bilgisayar adını ve yönetici şifresini yapılandırmak için VmConfig aracıyla yapılandırılan yanıt dosyasını ve ağ adaptörlerini kullanabilirsiniz (yanıt dosyasında parametreleri tanımlamak oldukça zordur). Diğer taraftan, Windows Server 2008 ve Windows Vista yanıt dosyasıyla tamamen yapılandırılabilir (bu kılavuzda anlatılmaktadır); böylece bu bölüm göz ardı edilebilir.
    • computerName – “Bilgisayar adı değiştir” VmConfig görevinin VPS denetimi sırasında yürütülüp yürütülmeyeceğini belirler. “true” görevin yürütüleceği anlamına gelir;
    • administratorPassword – “Yönetici şifresi değiştir” VmConfig görevinin VPS denetimi sırasında yürütülüp yürütülmeyeceğini belirler. “true” görevin yürütüleceği anlamına gelir;
    • networkAdapters – “Ağ adaptörleri ayarla” VmConfig görevinin VPS denetimi sırasında yürütülüp yürütülmeyeceğini belirler. “true” görevin yürütüleceği anlamına gelir;


Örnek “index.xml”:

<?xml version="1.0"?>

<items>

 

  <!-- Windows Server 2008 -->

  <item path="Windows-Server-2008-STD-x64"

        diskSize="10"

        remoteDesktop="true">

    <name>Windows Server 2008 Standard 64-bit</name>

    <description></description>

    <provisioning>

      <sysprep file="\Windows\System32\sysprep\unattend.xml"/>

      <sysprep file="\Windows\Setup\Scripts\rename-connections.vbs"/>

    </provisioning>

  </item>

 

  <!-- Windows Server 2003 -->

  <item path="Windows-Server-2003-ENT-x86"

        diskSize="8"

        remoteDesktop="true">

    <name>Windows Server 2003 Enterprise 32-bit</name>

    <description></description>

    <provisioning>

      <sysprep file="\Sysprep\sysprep.inf"/>

      <sysprep file="\Windows\Setup\Scripts\rename-connections.vbs"/>

      <vmconfig networkAdapters="true" />

    </provisioning>

  </item>

 

  <!-- Windows XP -->

  <item path="Windows-XP-Home">

    <name>Windows XP Home Edition</name>

    <description></description>

    <provisioning>

      <sysprep file="\Sysprep\sysprep.inf"/>

      <vmconfig networkAdapters="true" />

    </provisioning>

  </item>

 

</items>

index.xml dosyası oluşturulurken şunlara dikkat edin:

  • Dosya uzantısı “.xml” olmalıdır. Windows uygulamasında yeni bir .txt dosyası oluşturulurken bilinen uzantı gizlenebilir, bu nedenle yanlış anlama söz konusu olabilir.
  • Dosya kodlama. Şablon adları ve açıklamalarında çeşitli ulusal karakterlerin kullanılabilmesi için UTF-8 tavsiye edilir. Kodlamayı, Not Defteri uygulaması “Farklı kaydet...” iletişim penceresinde seçebilirsiniz.
