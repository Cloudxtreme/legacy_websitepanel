WSP Hosted BlackBerry Solution Pre-Deployment Tasks

Translations:

  • tr-TR

Assumptions

This document assumes that:

  • All background infrastructure (Windows Servers, Active Directory, DNS, Exchange severs and BlackBerry Enterprise Server) is configured correctly. As a minimum Windows event logs, Support tools utilities (such as dcdiag and netdiag) do not contain critical errors. As a maximum - all services are deployed according to Microsoft and Research in Motion guides and carefully documented.
  • BlackBerry Enterprise Server is deployed according to "BlackBerry Enterprise Server for Microsoft Exchange Installation Guide".
  • Also this document assumes that Exchange part of the system is already managed by WSP according to the appropriate WSP documentation.
  • BRK01 - BES Recourse Kit Server (with deployed BlackBerry Enterprise Server User Administration Tool on it) in our example.

General ideas

Two points should be achieved in Hosted BlackBerry environment managed by WSP:

  • Organization isolation - customers from one hosted organization should not see others when using BlackBerry devices. This is achieved by following BlackBerry manuals for Hosted BlackBerry Enterprise Server environment, and using special organization-unique AD attributes set by WSP.
  • Delegated management - end users should manage their BlackBerry accounts by themselves using WSP. This is achieved by deploying BlackBerry Enterprise Server Resource Kit tool and executing commands based on those tools using WSP web interface.

Achieving organization isolation

  • Download appropriate version of "BlackBerry Enterprise Server for Microsoft Exchange Installation Guide" from here http://na.BlackBerry.com/eng/support/docs/subcategories/?userType=2&category=BlackBerry+Enterprise+Server+for+Microsoft+Exchange
  • Open "BlackBerry Enterprise Server for Microsoft Exchange Installation Guide" document. Follow the steps according to "Configuring how users search for email addresses in a Hosted BlackBerry Enterprise Server environment" section. In "Configure the Hosted BlackBerry Enterprise Server to search for email addresses using a custom LDAP field" section set LDAPCompanyField to extensionAttribute1.

Deploying BlackBerry Enterprise Server Resource Kit

  • Download appropriate version (the same as your BES version) of BlackBerry Enterprise Server Resource Kit and related documentation from here http://na.BlackBerry.com/eng/support/server_resourcekit.jsp. Note that for WSP Hosted BlackBerry Solution you need to follow only "BlackBerry Enterprise Server User Administration Tool" part of this kit and documentation related to this tool only.
  • Deploy BlackBerry Enterprise Server User Administration Tool service and BlackBerry Enterprise Server User Administration Tool client according to "User Guide for the BlackBerry Enterprise Server User Administration Tool" document. They should be deployed on servers according to your environment diagram created on planning stage.
  • Test BlackBerry Enterprise Server User Administration Tool (besuseradminclient.exe) deployment by executing some test command. You can find usage samples in the "User Guide for the BlackBerry Enterprise Server User Administration Tool" document. Please ensure that test command runs successfully before moving to the next steps.
