WSP Barındırmalı CRM Çözümü Kurulum Görevleri

WSP Sunucu hesabı gereksinimleri

WSP Sunucu web sitesi anonim erişimi ve WSP Sunucu uygulaması havuz kimliği için WSP Sunucu hesabı alan hesabı olmalıdır. Bu hesabı CRM kurulumu tarafından oluşturulan PrivUserGroup'a ve SQL Server üzerindeki yerel Yöneticiler grubuna ekleyin. Bu hesap daha önceden Alan Yöneticileri grubuna eklenmişse (örneğin, Barındırılan Kurumlar için), SQL server üzerindeki yerel Yöneticiler grubuna eklemeye gerek yoktur.

 

"Active Directory Ayarları"nı yapma

Barındırılan CRM modülünün uygulanacak tüm WSP Sunucuları için "Active Directory Ayarları" gereksinimleri, Barındırılan Kurumların uygulanacağı WSP Sunucusundakilerle aynıdır.

 

Barındırılan CRM Hizmeti Kurma

Sunucuya Harici IP ekleme. Bu, CRM web sitesini barındıran AppServer'ın halka açık IP adresi olmalıdır. NAT tabanlı altyapı kullanılıyorsa CRM Web Sitesinin yayınlanacağı NAT sunucusunun harici IP adresini kullanın. Bu, istemcinin CRM siteleri için DNS kayıtları oluşturmak amacıyla kullanılacaktır. WSP ile DNS'yi yönetmeyecekseniz bile bu adresi ekleyin.

 

"İS" hizmetinin en az bir WSP sunucusunda uygulandığından emin olun. Klasik web barındırmayla birlikte Sanal Sunucu ve Barındırma planı oluşturacaksanız web sitelerini barındıracak WSP Sunucusundan "İS" hizmetini ekleyin. Sadece CRM veya karma WSS/Exchange/CRM ortamı söz konusuysa herhangi bir sunucuyu kullanın. Müşterileriniz WSS Sitesi koleksiyonunun yedeği için Barındırma Alanı klasörlerini kullanabilir. Ayrıca gelecekteki Exchange posta kutusu yedeği ve CRM kurumları yedek özellikleri de Barındırma Alanı klasörlerini kullanacaktır. "Barındırma Alanları Klasörü" dosya yolunu dikkatli bir şekilde seçin (yeterli boş alan olmalıdır; örneğin, programlar değil veri dosyalarının bulunduğu fiziksel diski seçin).

 

"Barındırılan Kurumlar" modülünün WSP Sunucularının herhangi birinde uygulandığından emin olun.

"Barındırılan CRM " hizmetinde hizmet ayarlarını belirtin (bunların çoğu, daha önceden CRM uygulamasında kullanılan CRM kurulum XML dosyasından alınabilir):

SQL Server. CRM SQL arka uç sunucu ana bilgisayar adı. Kurulum XML dosyasındaki "SqlServer" etiketinden kopyalanabilir.

Rapor URL adresi. CRM SQL arka uç sunucu Raporlama Hizmetleri URL adresi. Kurulum XML dosyasındaki "Rapor URL Adresi" etiketinden kopyalanabilir.

IFD Web Uygulaması Kök Alanı. Kurulum XML dosyasındaki "webapplicationrootdomain" etiketinden kopyalanabilir. Ya da IFD Uygulama Aracını çalıştırabilir ve oradan kopyalayabilirsiniz.

IFD Kök Alan Şeması. Kurulum XML dosyasındaki "rootdomainscheme" etiketinden kopyalanabilir. HTTPS kullanılması tavsiye edilir. Ya da IFD Uygulama Aracını çalıştırabilir ve oradan kopyalayabilirsiniz.

CRM Web Sitesi IP Adresi. Açılır listeden sunucu IP adresini seçin. Bu, harici istemcilerin bağlanacağı CRM Web Sitesini barındıran AppServer'ın Harici IP adresi olmalıdır.

CRM Web Sitesi portu. Bu alanı, sadece CRM Web Sitesi için HTTP veya HTTPS için standart olmayan (80 veya 443 harici) portlar kullanıyorsanız doldurun.

AD Uygulama Kök Alanı. IFD Uygulama Aracını çalıştırın ve “AD App Kök Alanı”nı kopyalayın

 

“AD Alan Şeması” IFD ayarlarında HTTPS değil HTTP olarak ayarlanmalıdır; aksi takdirde WSP CRM kullanıcı yönetimi çalışmayabilir. Doğru IFD uygulamasında dışarıya açılmayacağı için bu güvenliği etkilemeyecektir.

 

Sanal Sunucu Oluşturma

Barındırma planları sunucuları haricindeki sunucuların kullanılması tavsiye edilir. Bu, esnekliği büyük ölçüde arttırır ve mevcut müşterilere hizmet veya kaynak ekleme sırasındaki sorunları önlemenizi sağlar

Hedef sunucu olarak "Barındırılan CRM" sunucusunu seçin.

Uyguladığınız sunucudan "Barındırılan Kurumlar" hizmetini seçin.

Herhangi bir sunucudan "İS" hizmetini seçin. Klasik web barındırmayla birlikte Sanal Sunucu ve Barındırma planı oluşturacaksanız sadece web sunucusundan "İS" hizmetini kullanın.

Opsiyonel olarak, Barındırılan CRM ve Exchange'i tek bir Barındırma Planında kullanacaksanız veya Eklentilerle daha sonra müşterilere Exchange özellikleri ekleyecekseniz Exchange 2007 sunucusundan Posta Kutusu göreviyle "Exchange 2007" hizmetlerini seçin (daha muhtemeldir).

Opsiyonel olarak, Barındırılan CRM ve diğer hizmetleri tek bir Barındırma Planında kullanacaksanız veya Eklentilerle müşterilere diğer hizmetleri ekleyecekseniz "Barındırılan SharePoint" veya "Web" gibi hizmetleri seçin.

Opsiyonel olarak, WSP yönetimli DNS sunucusu kullanacaksanız uygun sunucudan "DNS" hizmetini seçin. Bu hizmet opsiyoneldir


Barındırma Planı Oluşturma

Karma CRM/Exchange Barındırma Planı oluşturuyorsanız planı bu belgeye uygun şekilde oluşturun ve daha sonra "Barındırılan CRM" seçeneğini ekleyin.

 

Saf CRM Barındırma Planı oluşturuyorsanız planı bu belgeye uygun şekilde oluşturun, "Exchange 2007" seçeneğini eklemeyin ve "Barındırılan CRM" seçeneğini ekleyin.

 

CRM Kurumu oluşturma

Bu noktada, Barındırma Planı oluşturulduktan sonra aşağıdaki adımların gerçekleştirilmiş olduğunu varsayıyoruz:

- WSP Kullanıcısı, barındırma sağlayıcı tarafından oluşturulmuş

- Bu WSP kullanıcısı için Barındırma Alanı (daha önceden oluşturulan Barındırma Planına göre) barındırma sağlayıcı tarafından oluşturulur

- Bu Barındırma Alanı içindeki Barındırma Kurumu, barındırma sağlayıcısı veya barındırma müşterisi tarafından oluşturulur

- Barındırma sağlayıcısı veya barındırma müşterisi tarafından Barındırılan Kurumda en az bir kullanıcı (gelecekteki CRM Kurumu yöneticisi) oluşturulur

Aşağıda CRM Kurumunu oluşturmak için barındırma müşterilerinin yapması gereken adımlar verilmiştir:

- Barındırılan Kurumlar > Müşteri Kurumu > CRM > CRM kurumu'na gidin

- Mevcut kurum kullanıcıları listesinden CRM kurumu için Yöneticiyi seçin

- Uygun Döviz ve Karşılaştırmayı seçin

- "Oluştur" düğmesine basın.

- CRM Kurumu portalını bir tarayıcıda açmayı deneyin. Yöneticinin temel e-postasını (UPN) kullanıcı adı olarak kullanın.
