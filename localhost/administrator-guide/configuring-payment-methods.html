Configuring Payment Methods

Translations:

  • tr-TR

Credit Card Payments

Here you can set a display name of the CC payment method and your current payment provider settings in order to be able accept payments online.  The following credit card payment providers are available in the current version: PayPal Pro and Authorize.Net. Once you specified all provider settings click “Save Settings” button. If you would like to stop accept online payments by credit card click “Disable” button (this applies only if payment method is already activated).  

Here is the fields description on the page:

Display Name – display name of the payment method in the "Select Payment Method" area on order checkout screen;

Accept Payments Using - credit cards payments acceptance provider;

Once you've selected payment provider some additional fields will be displayed. After all of fields are filled with the values click "Save Settings" button to save changes you made.

Accept Credit Card Online Payments by Authorize.Net

Please find below description for each particular field on this screen.

Username – your Authorize.Net account username;
Transaction Key – your current transaction key;
MD5 Hash Value – MD5 hash value can be taken from your Authorize.Net Account/Settings/MD5-Hash screen;
Demo Account – tick this checkbox if you use test ANET environment, in production tick it off;
Merchant Email – merchant email;
Send Confirmation – specifies whether ANET should send email notifications to the merchant email above;
Live Mode – specifies whether payments will be live or demo (in conjunction with Demo Account checked on ANET generates a demo transaction ID that similar to the production ones);

Accept Credit Card Online Payments by PayPal Pro

Please find below description for each particular field on this screen.

Username – your PayPal Pro account username;
Password – generated API password by PayPal;
Signature – generated API Signature by PayPal;
Live Mode – specifies whether payments will be live or demo;

Where I can find or obtain Signature field value?

Please follow this PayPal guide to generate API Signature before proceeding with the further plug-in configuration https://www.paypal.com/IntegrationCenter/ic_certificate.html (API Signature section).

2Checkout Payments

Please find below description for each particular field on this screen.

Display Name – display name of payment method when user select payment method during order checkout;

Secret Word - secret word you entered on 2CO side for your merchant account. Open merchant account home page on the 2CO side, click "Settings" link for "Look and Feel" and then after "Look and Feel" screen will appear scroll to the bottom of the page. Copy 2CO Secret Word string and paste it to the corresponding field in 2CO payment method settings screen;
2CO Account - 2CO merchant account unique identificator. Open merchant account home page on the 2CO side and copy if from the page title "Account xxxxxxx Homepage" where xxxxx is your vendor number;
2CO Currency - select a currency that you've specified in your bank account settings on 2CO side;
Live Mode - enables or disables demo 2ChekcOut transaction mode;
Fixed 2CO Cart - enables or disables "Quantity" field on the 2CO side. It is recommended to tick this checkbox on;
Once you specified all provider settings click “Save Settings” button. If you would like to stop accept online payments by 2Checkout click “Disable” button (this applies only if payment method is already activated).

PayPal Account Payments

Please find below description for each particular field on this screen.
Display Name – display name of payment method when user select payment method during order checkout;
Business - PayPal account username that you are using to login through website;
Live Mode - if you wish to accept live payments tick this checkbox on. If you wish to perform demo transactions leave it unchecked;
Once you specified all provider settings click “Save Settings” button. If you would like to stop accept online payments by PayPal account click “Disable” button (this applies only if payment method is already activated).

Offline Payments

Please find below description for each particular field on this screen.
Display Name – display name of payment method when user select payment method during order checkout;
Transaction ID Format (Payment Prefix) – transaction id format string for offline payments. The following variables can be used here:
yyyy – resolves as full year, for example 2007;
yy – resolves as short year, for example 07 (2007);
MM – resolves as month number, for example 05;
dd – resolves as day number, for example 25;
[INVOICE_ID] – resolves as invoice id, for example 58;
Let’s consider an example transaction id format DNP-yyyy/MM/dd#[INVOICE_ID] will be compiled is as follows: DNP-2007/05/25#58.
Once you specified all provider settings click “Save Settings” button. If you would like to stop accept payments by Offline Payment Gateway click “Disable” button (this applies only if payment method is already activated).
