Installing WebsitePanel Components

Translations:

  • tr-TR

WebsitePanel application has truly-distributed architecture and consists of three separate applications (components):

  • WSP Portal – user interface part of WebsitePanel
  • WSP Enterprise Server - provisioning framework comprising all application business logic.
  • WSP Server – “agent” application to perform low-level operations on controlled server.

Please see this article for more information about how WebsitePanel works.

If you are planning to have a dedicated WSP control server then only WSP Portal and WSP Enterprise Server should be installed there. However, if this server also should be managed from WSP (for example control server is a web server in the same time) it must have WSP Server installed.

Before installing WSP Enterprise Server install SQL Server 2005 or 2008 Express edition for internal WSP Enterprise Server meta-base. Make sure you have “Mixed” security mode enabled when both Windows and SQL Server authentications are allowed.

Please see this guide for more information about installing WebsitePanel components.
