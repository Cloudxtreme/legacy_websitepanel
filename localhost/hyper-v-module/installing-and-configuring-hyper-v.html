Enabling Remote Management

If you plan to manage from WSP remote (where WSP Server is not installed or cannot be installed on virtualization server) Windows Server 2008 Server Core with Hyper-V role or Hyper-V Server 2008 then you may use “HVRemote” tool to correctly setup Hyper-V remote management: http://code.msdn.microsoft.com/HVRemote

Please look at required configuration steps (see below):

  • Hyper-V hosts and “Management Server” should be joined to the same domain.
  • Hyper-V hosts are managed from a single WSP Server installed on “Management Server”.
  • WSP Server is not installed on Hyper-V server
  • WSPServer account must be a member of "Domain Admins" AD group and local "Administrators" groups on all controlled Hyper-V hosts
  • One more important thing is that you have to enable firewall rules on the Hyper-V host server for WMI (see below):

netsh advfirewall firewall set rule group="Windows Management Instrumentation (WMI)" new enable=yes

Do not copy-paste the commands from this page or another help sources into your server core console, because copy-pasted commands will give you an error message: “Group cannot be specified along with other identification conditions”.

For deepest understanding, how to configure the Hyper-V role for remote management on a full and Server Core installation of Windows Server 2008, please read this article http://technet.microsoft.com/en-us/library/cc794756(WS.10).aspx.

Checking Windows Updates

Please make sure you have the following KB950050 update installed on the Virtualization server:

http://www.microsoft.com/downloads/details.aspx?FamilyId=F3AB3D4B-63C8-4424-A738-BADED34D24ED&displaylang=en

You could see the list of installed Windows updates here: Control Panel -> Programs and Features -> Installed updates.

More info about KB950050 update: http://support.microsoft.com/kb/950050
