WSP Barındırmalı SharePoint Çözümü Ön Uygulama Görevleri

Kabuller

Bu belge arkaplan altyapısının (Windows Servers, Active Directory, WSS, DNS, vb) hatasız olarak yapılandırıldığını kabul eder. Minimum olarak Windows olay kayıtları, Destek araçları yardımcı programlarında (dcdiag ve netdiag gibi) kritik hatalar artık gösterilmektedir. Maksimum olarak tüm hizmetler Microsoft talimatlarına uygun olarak uygulanmıştır ve özenli bir şekilde belgelendirilmiştir; dolayısıyla hata ve uyarı bulunmamaktadır.

 

WSS 3.0, Active Directory ortamındaki "sunucu grubu" senaryosuna göre uygulanır.

 

Bu makaledeki örnekler, standart port (443) kullanan SSL korumalı SharePoint siteleri içindir. Ancak, SSL korumalı ve standart dışı bir port (8080 gibi) için ayarlanabilir. Bununla birlikte, güvenlik için sadece SSL korumalı SharePoint sitelerinin kullanılmasını tavsiye ederiz.

 

Genel fikirler

WSP Barındırmalı SharePoint hizmeti SharePoint Web Uygulamasının (biz "Kök Web Uygulaması" olarak adlandırıyoruz) üstüne ana bilgisayar tarafından adlandırılmış Site Koleksiyonlarını oluşturur. Bu yöntem, tek bir sunucuda binlerce SharePoint sitesi barındırmanıza olanak verir.

 

Kök Web Uygulamasını el ile oluşturabilir ve daha sonra Barındırılan SharePoint hizmetini bunu kullanacak şekilde ayarlayabilirsiniz. Birden fazla Kök Web Uygulaması ve Barındırılan SharePoint hizmeti kullanabilirsiniz; örneğin SSL ve SSL dışı veya joker karakter sertifikası veya bu sertifika olmadan. Exchange Depolama Grupları/Veritabanları ve buna bağlı farklı Barındırılan Exchange hizmetlerine benzer bir şey. Ancak birçok durumda bir SSL korumalı Kök Web Uygulamasına sahip olmak tamamıyla yeterli.

 

Kök web uygulaması oluşturma

WSS ön uç sunucusunda WSP açık Yönetici araçları > SharePoint 3.0 Merkezi Yönetimi > Uygulama Yönetimi > Web Uygulamasını Oluştur veya Genişlet > Yeni bir web uygulaması oluştur ile yöneteceksiniz.

 

Seçebileceğiniz varsayılan dışındaki ayarlar (tüm bu ayarlar zorunlu değildir, kendi vizyonunuza göre başka ayarlar seçebilirsiniz).

 

- Açıklama. Anlamlı bir şeyler. Örneğimizde "Barındırma Kökü - SSL"

- Port. Kullanmaya karar verdiğiniz herhangi bir port. Örneğimizde " 443" (SSL korumalı Web Uygulaması oluşturacağız ve klasik SSL portu kullanacağız)

- Anonime izin ver: Size bağlı. Örneğimizde "Evet" (Tüm yeni oluşturulan Site Koleksiyonlarına anonim erişim açmayacak ancak site sahiplerinin gerekirse Site Koleksiyonlarına anonim açmasına olanak verecek).
- Güvenli Yuva Katmanı (SSL) kullanın. Size bağlı. Örneğimizde "Evet" (Barındırılan Exchange ile kullanılacağını varsayıyoruz; dolayısıyla kullanıcılar SharePoint'e Exchange posta kutularına bağlanırken kullandıkları e-posta/şifre kombinasyonuyla bağlanır; dolayısıyla bağlantı güvenli olacaktır).

- Uygulama havuzu adı. Size bağlı. Örneğimizde "Barındırma Kökü - SSL".

- Bu uygulama havuzu için bir güvenlik hesabı seçin. Size bağlı. Örneğimizde "Önceden tanımlanmış - Ağ Hizmeti". Ancak, güvenlik nedenleriyle ayrı bir hesap kullanmaya karar verebilirsiniz.

- Windows SharePoint Hizmetleri arama sunucusunu seçin. Kök Web Uygulamasını oluşturmadan önce aramanın başlatılması ve yapılandırılması ve yapılandırılmış arama sunucusunun buna bağlanması daha iyidir.

 

Değiştirilmemesi gereken varsayılan ayarlar.

- Kimlik denetimi sağlayıcısı. "NTLM" kullanın; aksi takdirde aramayla ilgili sorun yaşamanız muhtemeldir.

- Ayrıca nedensiz yere başka bir varsayılan ayarı değiştirmeyin.

 

Kök Web Uygulaması oluşturmayı sonlandırma

- Kök Web Uygulaması URL adresini not edin (ya da alandan kopyalayıp yapıştırın). Bu daha sonra WSP ayarları için kullanılacaktır.

- "Tamam" düğmesine basın ve Web Uygulaması oluşturulana kadar bekleyin.

- Komut satırında iisreset /noforce'u çalıştırın.


SSL sertifikasıyla kök Web Uygulamasının güvenliğini sağlama

Bu adım isteğe bağlı olmakla birlikte şiddetle tavsiye edilir.

 

- IIS, kök Web Uygulaması web sitesine gidin ve sertifika talebini oluşturun. Müşterileriniz tahmin edilebilir olmayan farklı URL adresleriyle Site Koleksiyonları oluşturacağı için bu sertifika için herhangi bir Genel Ad kullanabilirsiniz. Dolayısıyla her koşulda tarayıcılarda sertifikayla ilgili hatalar/uyarılar alacaklardır.

- CA'dan SSL sertifikası alın. Para tasarrufu için dahili CA'nızı kullanabilirsiniz. Site Koleksiyonlarının URL adresleri tahmin edilemez olsa da kullanıcılar VeriSign'den sertifika satın alsanız bile tarayıcılarında sertifikayla ilgili hatalar/uyarılar alacaktır.

- Sertifikayı kök Web uygulaması web sitesine kurun.

- Kök Web Uygulaması web sitesi istemcilerini sadece SSL kullanacak şekilde kısıtlayın. Web sitesi özelliklerini açın > Dizin Güvenliği > Güvenli iletişim > Düzenle > Güvenli kanal iste (SSL), 128 bit şifreleme iste.

Tarayıcılarda sertifikayla ilgili hataları/uyarıları önlemenin bazı yöntemleri bulunmaktadır.

 

İlk olarak her müşteri için kök Web Uygulaması oluşturabilirsiniz, ancak WSS'nin kaynak açısından çok yoğun bir uygulama olması ve tek bir sunucu üzerinde birçok Web Uygulaması çalıştırabileceğiniz için durum böyle değildir.

 

İkinci olarak joker karakter sertifikası (*.providerdomain.com) satın alabilir ve müşterilerinizin client.providerdomain.com gibi URL adreslerine sahip Site Koleksiyonları oluşturmasını sağlayabilirsiniz. Ancak müşterilerin çoğu tarayıcılarda sertifikayla ilgili hatalar/uyarılar ve team.client.com gibi kullanıcı URL adresleri görmeyi tercih etmektedir.

 

Kök Web Uygulaması düzeyinde AD taramasını devre dışı bırakma

Seçilen insanları kısıtlayın, böylelikle kullanıcılar SharePoint ile başka bir kurumun kullanıcılarını çözümleyemeyecektir.

Komut satırını açın, dizini WSS BIN klasörü olarak değiştirin (varsayılan olarak C:\Program Files\Common Files\Microsoft Shared\Web Server Extensions\12\BIN) ve aşağıdakini çalıştırın:

stsadm -o setproperty -pn peoplepicker-onlysearchwithinsitecollection -url https://wspwssdemolab01/ -pv yes

 

Burada "-url", daha önceden yazdığınız Kök Web Uygulaması URL adresidir; örneğimizde, "https://wspwssdemolab01/". Https için standart olduğu için bu URL adresinde port numarası atlanır. Standart olmayan (80 veya 443 değil) port kullanıyorsanız URL adresine port numarasını girmeyi unutmayın.

 

Özelliğin "-o getproperty" stsadm swith ile düzgün şekilde ayarlanmış olup olmadığını kontrol edin. Örneğin:

stsadm -o getproperty -pn peoplepicker-onlysearchwithinsitecollection -url https://wspwssdemolab01/

Aşağıdaki sonuç gösterilmelidir:

<Property Exist="Yes" Value="yes" />

 

Barındırılan Kurumlar modülünü uygulama

Daha önceden uygulanmışsa bu adımı atlayın (örneğin, HES için).

Hala uygulanmamışsa, bu makalede açıklanan adımları uygulayın. Bu modülü ya WSS sunucusuyla aynı bilgisayarda veya WSP Sunucusu kurulu olan başka bir alan sunucusunda uygulayabilirsiniz.
