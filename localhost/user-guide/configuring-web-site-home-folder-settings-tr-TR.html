Web Sitesi Ana Sayfası Klasör Ayarlarını Yapılandırma

Web sitesi özellikleri sayfası, “Ana Sayfa Klasörü” sekmesini açın:

 

Ana sayfa dizin konumu

Web sitesini barındırma alanı içindeki bir klasöre yönlendirmek için “Barındırma Alanınızdaki Klasör” öğesini seçin. Barındırma planında izin verilmişse varsayılan “wwwroot” klasöründen web sitesi kök klasörünü değiştirebilirsiniz.

Web tarayıcısı gibi istemci uygulamasından gelen tüm talepleri aşağıdaki kutuya bir URL adresi girerek başka bir web sitesine veya sanal dizine yönlendirmek için “URL Adresine Yönlendirme”.

 

Güvenlik ayarları

Web sitesi kullanıcılarının sunucunuzda etkinleştirilmiş dizine dosyalar ve ilgili özelliklerini yüklemesini veya Yazma izinli bir dosyada içeriği değiştirmesini sağlamak için “Yazma İzinlerini Etkinleştir” seçeneğini işaretleyin. Yazma erişimi, sadece web tarayıcı HTTP 1.1 protokolü standardının PUT özelliğini desteklediğinde sağlanır.

Web sitesi ziyaretçilerinin söz konusu sitedeki dosyalar ve alt dizinlerin hiper metin listesini görmesini sağlamak için “Dizin Taramayı Etkinleştir” seçeneğini işaretleyin.

 

Web sayfalarının geçerli dizinin üst düzey dizinine görece yollar (“..” sözdizimini kullanan yollar) kullanmasını sağlamak için “Üst Düzey Dosya Yollarını Etkinleştir” seçeneğini işaretleyin. Bu seçenek, eski ASP uygulamaları için gerekebilir.

 

Kimlik denetimi

Bu web sitesine anonim bağlantı sağlamak için “Anonim Erişim Sağla” seçeneğini işaretleyin. Normalde, bu seçenek web sitelerinin birçoğu için etkindir.

 

Web sitesi kullanıcılarının web sitesine erişmesi için kimlik bilgilerini girmesini zorunlu hale getirmek için “Tümleşik Windows Kimlik Denetimini Etkinleştir” seçeneğini işaretleyin. Windows kimlik denetimi kullanıcı adı ve şifrenin, karma biçiminde ağa gönderilmesini sağlar. Bu, güvenli bir kimlik denetim biçimi sağlar. Normalde, bu seçenek web sitelerinin birçoğu için devre dışıdır.

 

Web sitesi kullanıcılarının web sitesine erişmesi için kimlik bilgilerini girmesini zorunlu hale getirmek için “Basit Kimlik Denetimini Etkinleştir” seçeneğini işaretleyin. Basit kimlik denetimi, HTTP spesifikasyonunun bir parçasıdır ve birçok tarayıcı tarafından desteklenmektedir; ancak kullanıcı adı ve şifre, şifrelenmemiştir ve güvenlik riskleri taşıyabilir. Normalde, bu seçenek web sitelerinin birçoğu için devre dışıdır.

 

Varsayılan Belgeler

Varsayılan belgeler, bir tarayıcı talebi bir belge adı belirtmediğinde web sunucusu tarafından varsayılan bir belgeyi tespit etmek için kullanılır. Üstten alta doğru bir arama sırasında varsayılan belgeleri belirtin.
